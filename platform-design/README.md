## 📚 核心文档

本目录包含平台的核心设计文档，采用**三层文档体系**：

### 1. 业务方案层

📘 **[AUTO_RD_PLATFORM_SOLUTION_MERMAID.md](./AUTO_RD_PLATFORM_SOLUTION_MERMAID.md)**
- **规模**: 2097行，58个mermaid图表，11章
- **内容**: 
  - 问题域与业务场景
  - 平台业务架构（四层架构）
  - 研发价值流设计（S1-S9）
  - 需求与资产模型（三层模型）
  - 能力架构与核心功能（C0-C7）
  - 流程定义详解
  - 角色协同与职责
  - 平台核心价值
- **用途**: 方案评审、架构设计、高层决策

### 2. 功能清单层

📄 **[全量功能清单_C0-C7.md](./全量功能清单_C0-C7.md)**
- **规模**: 636行，13章，179个功能
- **内容**:
  - C0-C7全部179个功能详细定义
  - 每个功能包含：编号、名称、描述、输入、输出、角色、场景、价值流阶段
  - 5个附录：
    - 附录A: 按角色分类索引（11个角色）
    - 附录B: 按价值流阶段分类索引（9个阶段）
    - 附录C: 按业务场景分类索引（4个场景）
    - 附录D: 功能依赖关系（6条关键链）
    - 附录E: 使用指南（3个实用场景）
- **用途**: 实施参考、产品Backlog规划、开发任务分解、验收测试

### 3. 设计方法层

📖 **[平台设计方法论.md](./平台设计方法论.md)**
- **规模**: 582行
- **内容**:
  - 价值流优化方法（瓶颈识别、改进策略）
  - 需求分解方法（四层模型、SSTS模板）
  - 资产复用决策（评估维度、ROI计算）
  - 流程设计原则（6大要素、状态管理）
  - 性能度量体系（9大指标）
- **用途**: 设计指导、最佳实践参考

## 🎯 文档使用指南

### 场景1: 方案评审

**推荐文档**: AUTO_RD_PLATFORM_SOLUTION_MERMAID.md

**使用流程**:
1. 查看第1章「问题域与业务场景」- 理解业务挑战
2. 查看第2章「平台业务架构」- 理解四层架构
3. 查看第3章「研发价值流设计」- 理解S1-S9价值流
4. 查看第5章「能力架构与核心功能」- 理解C0-C7能力域
5. 查看第11章「平台核心价值」- 理解平台价值主张

### 场景2: 产品Backlog规划

**推荐文档**: 全量功能清单_C0-C7.md

**使用流程**:
1. 查看「附录E: 使用指南」第1节 - 学习Backlog规划方法
2. 将8个能力域作为Epic
3. 将179个功能作为Feature
4. 按「附录B: 按价值流阶段分类索引」排序
5. 按优先级分配到PI

### 场景3: 开发任务分解

**推荐文档**: 全量功能清单_C0-C7.md

**使用流程**:
1. 查看「附录E: 使用指南」第2节 - 学习任务分解方法
2. 选择功能（如C1-F06 PRD编写）
3. 查看功能定义（输入、输出）
4. 拆解为可执行Task
5. 估算工作量

### 场景4: 验收测试

**推荐文档**: 全量功能清单_C0-C7.md

**使用流程**:
1. 查看「附录E: 使用指南」第3节 - 学习验收测试方法
2. 为每个功能设计测试用例
3. 验证输入→功能→输出
4. 检查角色权限
5. 验证价值流集成

### 场景5: 设计指导

**推荐文档**: 平台设计方法论.md

**使用流程**:
1. 查看「价值流优化方法」- 识别瓶颈
2. 查看「需求分解方法」- 四层模型设计
3. 查看「资产复用决策」- 评估维度设计
4. 查看「流程设计原则」- 流程引擎设计
5. 查看「性能度量体系」- 度量指标设计
