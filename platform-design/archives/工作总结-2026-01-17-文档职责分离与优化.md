#### 6. 附录E: 使用指南（+86行）

**三大使用场景**:

##### 场景1: 产品Backlog规划

**步骤**:
1. 将8个能力域作为Epic
2. 将179个功能作为Feature
3. 按价值流阶段排序（S1→S9）
4. 按优先级分配到PI

**示例**:
```
Epic: C1-需求管理
├─ Feature: C1-F01~F04 (Epic管理) - P0
├─ Feature: C1-F05~F11 (Feature管理) - P0
├─ Feature: C1-F12~F17 (MR管理) - P1
└─ Feature: C1-F18~F25 (需求服务) - P2
```

##### 场景2: 开发任务分解

**步骤**:
1. 选择功能（如C1-F06 PRD编写）
2. 查看输入输出定义
3. 拆解为可执行Task
4. 估算工作量

**示例**:
```
功能: C1-F06 PRD编写
输入: Feature需求
输出: PRD文档

Task拆解:
├─ Task 1: 设计PRD编辑器UI (3SP)
├─ Task 2: 实现在线编辑功能 (5SP)
├─ Task 3: 实现文档上传功能 (3SP)
├─ Task 4: PRD版本管理 (5SP)
└─ Task 5: PRD评审流程集成 (3SP)
```

##### 场景3: 验收测试

**步骤**:
1. 为每个功能设计测试用例
2. 验证输入→功能→输出
3. 检查角色权限
4. 验证价值流集成

**示例**:
```
功能: C3-F03 PI Planning创建

测试用例:
1. 前置条件: 版本计划已创建、团队列表已维护
2. 输入: 版本计划、团队列表
3. 操作: PM/TPM创建PI Planning
4. 预期输出: PI Planning记录
5. 后置验证: 可进行PI目标设定(C3-F04)
```

**价值**: 
- 提供实施指导
- 降低学习成本
- 加速项目落地
