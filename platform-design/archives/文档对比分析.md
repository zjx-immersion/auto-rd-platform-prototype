## 六、新文档的核心创新

### 6.1 内容创新

#### 创新1: 四层需求模型（扩展）

**之前（三层）**:
```
Epic → Feature/SSTS → Module/MR → Task
```

**现在（四层+需求池）**:
```
需求池 (RequirementPool)
  ↓
Epic (只读控制、变更管理)
  ↓
Feature/SSTS (PRD双模式、批量评审)
  ↓
Module/MR (团队视角、批量操作)
  ↓
Task (自动拆解、看板管理)
```

**新增特性**:
- ✅ 需求池作为Epic源头
- ✅ Epic只读控制和变更管理
- ✅ PRD双模式（在线编辑+上传）
- ✅ SSTS批量评审（可查看PRD）
- ✅ 团队视角MR管理

#### 创新2: 流程驱动系统（新增）

**之前**: 无流程驱动设计

**现在**: 完整的流程引擎
```typescript
流程定义层
  ├─ 流程模板库
  ├─ 流程定义
  └─ 步骤定义

流程实例层
  ├─ 流程实例
  ├─ 流程上下文
  └─ 流程历史

流程引擎
  ├─ 启动引擎
  ├─ 步骤引擎
  ├─ 暂停/恢复
  └─ 完成引擎

预定义流程:
  ├─ PI Planning流程（10步）
  ├─ 需求拆解流程（7步）
  └─ Sprint规划流程（7步）
```

#### 创新3: 范围控制架构（新增）

**之前**: 无范围控制设计

**现在**: 4层范围控制
```
L1: 项目范围 → 项目规划、需求拆解、整体进度
L2: PI范围 → PI Planning、Feature分配、依赖管理
L3: Sprint范围 → Backlog规划、Task分配、日常执行
L4: Team范围 → 团队Backlog、人员分工、团队度量

特殊维度:
- 产品维度: 按产品过滤Feature
- 模块维度: 按模块团队过滤MR
```

#### 创新4: 能力域集成与联通（增强）

**之前**: 简单的能力域关系

**现在**: 完整的集成设计
```
端到端业务流集成:
  - sequenceDiagram（15步）
  - C0→C1→C2→C3→C4→C5→C6→C7→C0闭环

核心集成场景:
  - 场景1: 需求到开发的联通
  - 场景2: 资产复用的联通
  - 场景3: 质量反馈闭环

功能联通矩阵:
  - 9×9矩阵
  - 集成点、数据传递、触发条件

数据流集成:
  - 需求数据流
  - 规划数据流
  - 执行数据流
  - 质量数据流
```
