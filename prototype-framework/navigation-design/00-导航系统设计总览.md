## 一、导航系统概述

### 1.1 设计目标

Auto-RD平台的导航系统基于 **179个功能** 和 **9个价值流阶段**，旨在为用户提供清晰、高效、符合工作习惯的导航体验，支持：

- **流程驱动**: 按照研发流程引导用户完成工作（向导式步骤引导）
- **功能直达**: 快速访问常用功能和工具（功能分组菜单）
- **工作台聚合**: 个性化的工作台，聚合待办和关注内容（Widget系统）
- **智能搜索**: 全局搜索，快速定位任何内容

### 1.2 设计原则

1. **角色导向**: 根据用户角色智能推荐入口模式
2. **场景适配**: 不同场景自动切换最优入口模式
3. **灵活切换**: 三种模式可随时切换，互不干扰
4. **一致体验**: 统一的视觉语言和交互模式
5. **渐进式**: 从简单到复杂，从引导到自主

## 二、导航系统架构

### 2.1 三层导航结构

```
┌─────────────────────────────────────────────────────────────┐
│ 顶部导航栏 (Top Navigation)                                 │
│ [Logo] [工作台] [流程] [功能] [搜索] [通知] [用户]          │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│ 左侧导航栏 (Side Navigation)                                │
│ - 一级菜单                                                  │
│   - 二级菜单                                                │
│     - 三级菜单                                              │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│ 面包屑导航 (Breadcrumb)                                     │
│ 工作台 > 需求管理 > Feature管理 > ADAS-F001                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 三种导航模式

| 导航模式 | 适用场景 | 主要用户 | 特点 |
|---------|---------|---------|------|
| **流程驱动导航** | 按研发流程完成工作 | 所有角色 | 流程化、引导性强 |
| **固有功能导航** | 直接访问特定功能 | 高级用户 | 功能完整、层级清晰 |
| **工作台导航** | 个性化工作聚合 | 所有角色 | 个性化、效率高 |

## 五、核心设计要点

### 5.1 流程驱动导航 (模式1)

**核心理念**: 按照汽车研发的V模型流程组织导航

```mermaid
graph LR
    A[需求管理] --> B[资产管理]
    B --> C[开发管理]
    C --> D[测试管理]
    D --> E[发布管理]
    E --> F[运维管理]
```

**适用场景**:
- 新用户快速上手
- 按流程完成完整的研发任务
- 团队协作，确保流程规范

**详见**: [02-模式1-流程驱动导航设计.md](./02-模式1-流程驱动导航设计.md)

### 5.2 固有功能导航 (模式2)

**核心理念**: 按照功能模块组织导航，提供完整的功能访问

```
C0-工作台
C1-需求管理
  ├── Epic管理
  ├── Feature管理
  ├── SSTS管理
  └── MR管理
C2-资产管理
C3-开发管理
...
```

**适用场景**:
- 高级用户直接访问特定功能
- 功能管理和配置
- 数据查询和报表

**详见**: [03-模式2-固有功能导航设计.md](./03-模式2-固有功能导航设计.md)

### 5.3 工作台导航 (模式3)

**核心理念**: 个性化的工作台，聚合用户关注的内容

```
工作台
├── 我的待办 (按优先级排序)
├── 我的关注 (自定义关注)
├── 快捷入口 (常用功能)
└── 数据看板 (个人数据)
```

**适用场景**:
- 日常工作的起点
- 快速查看待办和关注
- 个性化工作空间

**详见**: [04-模式3-工作台导航设计.md](./04-模式3-工作台导航设计.md)

## 五、关键设计决策

### 6.1 导航模式切换

用户可以在三种导航模式之间自由切换：

```
顶部导航栏:
[工作台] [流程视图] [功能视图]
   ↓         ↓          ↓
 模式3     模式1      模式2
```

### 6.2 导航层级控制

- **最多三级**: 导航层级不超过三级，避免过深
- **扁平化**: 优先使用扁平化结构，减少点击次数
- **面包屑**: 始终显示面包屑，帮助用户定位

### 6.3 响应式设计

- **桌面端**: 完整的三层导航结构
- **平板端**: 可折叠的左侧导航
- **移动端**: 底部导航栏 + 抽屉式菜单

## 七、与页面设计的关系

### 7.1 导航与页面的映射

导航系统为页面设计提供访问路径：

```
导航路径: 需求管理 > Feature管理 > Feature列表
    ↓
页面设计: C1-F04-Feature列表.md
    ↓
页面URL: /requirements/features
```

### 7.2 页面跳转设计

每个页面设计文档都包含：
- **入口**: 从哪些导航路径可以访问
- **出口**: 可以跳转到哪些页面

**详见**: [05-页面跳转关系设计.md](./05-页面跳转关系设计.md)
