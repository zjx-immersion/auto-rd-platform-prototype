## 一、项目概述

### 1.1 项目定位

**面向智能驾驶、智能座舱、电子电器、底盘架构、新能源等领域的端到端研发协同平台**

### 1.2 核心目标

构建高效、透明、协同的端到端整车软件研发管理平台，支撑：
- **价值流驱动**: 9阶段价值流优化，从市场洞察到产品交付
- **领域模型**: DDD领域模型设计，5大核心领域
- **流程协同**: 8大核心流程，支撑全生命周期管理
- **资产复用**: AI驱动的资产推荐，复用率从30%→70%

### 1.3 技术栈

**前端**:
- Vue 3 + TypeScript + Element Plus
- Pinia (状态管理)
- Vue Router
- Mock数据模拟

**设计**:
- DDD领域模型
- Mermaid可视化
- 价值流驱动设计

### 2.2 核心设计理念

#### 1. 三层需求模型 + 工作项

**需求层**（3层）:
```
L1: Epic（用户价值需求）
  ↓
L2: Feature/SSTS（功能需求/软件系统技术规格）
  ↓
L3: Module/MR（模块需求）
```

**工作项**（非需求层）:
```
Task（工作项）
- 从MR转化而来
- 用于计划、开发、测试管理和跟踪
- 可完整溯源到需求链路源头
```

#### 2. 三层资产模型

```
Product（产品资产）
  ↓
Feature（功能资产）
  ↓
Module（模块资产，绑定Team）
```

#### 3. 九阶段价值流

```
S1市场洞察 → S2需求分解 → S3资产规划 → S4项目立项 → S5迭代开发
→ S6集成验证 → S7测试验收 → S8制品晋级 → S9产品交付
```

#### 4. 八大核心流程

```
1. PI Planning流程（10步）
2. 需求拆解流程（4步）
3. Sprint规划流程（5步）
4. Sprint执行与交付流程（承接PI Planning，2周闭环）
5. 特性需求评审流程（PRD评审+SSTS批量评审）
6. 产品需求版本管理与发布流程（版本规划+CCB变更）
7. 项目基线管理流程（车型节点基线+CCR/CCB）
8. 需求全生命周期变更管理流程（5阶段分层控制）
```

#### 5. 八大核心能力域（C0-C7）

```
C0: 领域项目管理（项目、版本、基线、多PI）
C1: 需求管理（Epic/Feature/SSTS/MR）
C2: 产品与资产管理（资产推荐、复用、评估）
C3: 规划协调（PI Planning、依赖、风险）
C4: 迭代执行（Sprint、Task、看板、团队协同）
C5: 测试验收（V模型验证、测试管理、质量门禁）
C6: DevOps交付（CI/CD、制品管理、灰度发布）
C7: 分析与治理（效能、质量、追溯分析）
```
