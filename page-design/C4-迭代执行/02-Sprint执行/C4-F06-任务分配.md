# C3-F06: 任务分配

> **功能编号**: C3-F06  
> **功能名称**: 任务分配  
> **所属模块**: C3-开发管理  
> **所属流程**: P4-团队迭代流程 (步骤3.1)  
> **优先级**: P0 (核心功能)  
> **设计日期**: 2025-01-15

---

## 一、功能概述

### 1.1 功能定位

任务分配是Sprint执行的第一步，负责将Task分配给具体的开发人员，明确任务责任人和计划时间。

### 1.2 核心价值

- ✅ 明确任务责任人
- ✅ 合理分配工作负载
- ✅ 优化资源利用
- ✅ 支持并行开发
- ✅ 提高开发效率

### 1.3 需求流转

```
Sprint → Backlog → Task → Assign → Development
                            ↑
                      P4步骤3.1 (当前)
```

### 1.4 用户角色

- **主要用户**: DL
- **协作用户**: Developer
- **查看用户**: PM、SM、All

## 三、分配规则

### 3.1 技能匹配

1. **技能要求**: 根据Task技能要求匹配开发者
2. **技能评分**: 计算开发者技能匹配度
3. **推荐开发者**: 推荐技能匹配度最高的开发者

### 3.2 负载均衡

1. **当前负载**: 统计开发者当前负载
2. **预测负载**: 预测分配后的负载
3. **负载平衡**: 避免开发者负载过高或过低

### 3.3 依赖关系

1. **前置依赖**: 考虑Task的前置依赖
2. **并行开发**: 支持无依赖的Task并行开发
3. **关键路径**: 识别关键路径上的Task

---

## 四、业务规则

### 4.1 分配规则

1. **一个Task一个人**: 每个Task只能分配给一个人
2. **技能匹配**: 开发者技能必须满足Task要求
3. **负载合理**: 开发者负载率建议70%-85%
4. **计划时间**: 必须指定计划开始和完成时间
