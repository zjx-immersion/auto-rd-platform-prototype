# C3-F11: 每日站会

> **功能编号**: C3-F11  
> **功能名称**: 每日站会  
> **所属模块**: C3-开发管理  
> **所属流程**: P4-团队迭代流程 (每日)  
> **优先级**: P1 (重要功能)  
> **设计日期**: 2025-01-15

---

## 一、功能概述

### 1.1 功能定位

每日站会是敏捷开发的核心实践，负责团队每日同步进度、识别障碍和调整计划。

### 1.2 核心价值

- ✅ 同步团队进度
- ✅ 识别障碍和风险
- ✅ 促进团队协作
- ✅ 快速调整计划
- ✅ 提高透明度

### 1.3 需求流转

```
Sprint → Daily Standup → Task Update → Progress Tracking
            ↑
      每日例行 (当前)
```

### 1.4 用户角色

- **主要用户**: Team (全体成员)
- **主持人**: SM
- **参与者**: DL、Developer、QA

## 三、站会规则

### 3.1 站会流程

1. **Sprint进度回顾**: 回顾Sprint整体进度
2. **成员汇报**: 每个成员汇报昨天、今天、障碍
3. **障碍识别**: 识别和记录障碍
4. **行动项确定**: 确定解决障碍的行动项

### 3.2 汇报内容

1. **昨天完成**: 昨天完成的Task和工作
2. **今天计划**: 今天计划完成的Task和工作
3. **遇到的障碍**: 遇到的问题和需要的帮助

### 3.3 时间控制

1. **站会时长**: 15分钟
2. **每人时长**: 2-3分钟
3. **准时开始**: 每天固定时间
