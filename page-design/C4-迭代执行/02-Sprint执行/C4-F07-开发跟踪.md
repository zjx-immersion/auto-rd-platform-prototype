# C3-F07: 开发跟踪

> **功能编号**: C3-F07  
> **功能名称**: 开发跟踪  
> **所属模块**: C3-开发管理  
> **所属流程**: P4-团队迭代流程 (步骤3.2)  
> **优先级**: P0 (核心功能)  
> **设计日期**: 2025-01-15

---

## 一、功能概述

### 1.1 功能定位

开发跟踪是Sprint执行流程的核心功能，负责实时跟踪任务状态、团队进度和Sprint健康度。

### 1.2 核心价值

- ✅ 实时跟踪任务状态
- ✅ 可视化团队进度
- ✅ 识别风险和阻塞
- ✅ 支持每日站会
- ✅ 提高团队协作效率

### 1.3 需求流转

```
Epic → Feature → PRD → SSTS → MR → Task → 开发跟踪
                                      ↑        ↑
                                   Sprint   P4步骤3.2 (当前)
```

### 1.4 用户角色

- **主要用户**: DL、Developer、SM
- **次要用户**: PM、QA
- **查看用户**: All

## 三、任务状态定义

### 3.1 任务状态流转

```
To Do → In Progress → Code Review → Testing → Done
  ↓         ↓            ↓            ↓
Blocked   Blocked      Blocked      Blocked
```

### 3.2 状态说明

| 状态 | 说明 | 操作权限 |
|-----|------|---------|
| To Do | 待开始 | 开发者可领取 |
| In Progress | 进行中 | 开发者可更新进度 |
| Code Review | 代码评审 | 评审者可评审 |
| Testing | 测试中 | QA可测试 |
| Done | 已完成 | 只读 |
| Blocked | 阻塞 | 需要解决阻塞问题 |

## 四、业务规则

### 4.1 任务更新规则

1. **每日更新**: 开发者每日更新任务进度
2. **状态流转**: 按照定义的状态流转
3. **工作日志**: 记录重要的工作进展
4. **阻塞标记**: 及时标记阻塞问题

### 4.2 风险识别规则

- **进度落后**: 实际进度 < 计划进度 20%
- **任务阻塞**: 阻塞时间 > 1天
- **负载不均**: 成员负载差异 > 30%

### 4.3 看板规则

- **WIP限制**: In Progress列最多3个任务/人
- **拉动式**: 完成一个任务后再拉取新任务
- **可视化**: 所有任务状态实时可见
