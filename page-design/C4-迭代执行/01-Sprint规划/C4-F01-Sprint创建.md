# C3-F01: Sprint创建

> **功能编号**: C3-F01  
> **功能名称**: Sprint创建  
> **所属模块**: C3-开发管理  
> **所属流程**: P4-团队迭代流程 (步骤2.1)  
> **优先级**: P0 (核心功能)  
> **设计日期**: 2025-01-15

---

## 一、功能概述

### 1.1 功能定位

Sprint创建是团队迭代的起点，负责创建新的Sprint周期，定义Sprint目标、时间范围和团队容量。

### 1.2 核心价值

- ✅ 建立迭代周期
- ✅ 明确Sprint目标
- ✅ 规划团队容量
- ✅ 设定交付预期
- ✅ 启动团队协作

### 1.3 需求流转

```
Epic → Feature → PRD → SSTS → MR → Sprint → Task
                                      ↑
                                  P4步骤2.1 (当前)
```

### 1.4 用户角色

- **主要用户**: PM、SM
- **协作用户**: DL
- **查看用户**: All

## 三、业务规则

### 3.1 Sprint周期规则

1. **标准周期**: 2周 (10个工作日)
2. **最短周期**: 1周 (5个工作日)
3. **最长周期**: 4周 (20个工作日)
4. **排除日期**: 自动排除周末和节假日

### 3.2 容量规划规则

1. **理论容量**: 人数 × 工作日 × 每日工作时长
2. **实际容量**: 理论容量 - 会议时间 - 休假 - 其他事务
3. **容量利用率**: 建议70%-85%
4. **容量预警**: <60%或>90%时提示

### 3.3 MR选择规则

1. **状态要求**: 必须是"已评审通过"状态
2. **依赖检查**: 检查MR依赖关系
3. **容量匹配**: 总工作量不超过团队容量
4. **优先级**: 优先选择P0级别MR
