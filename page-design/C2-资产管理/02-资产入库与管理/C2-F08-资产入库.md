# C2-F08: 资产入库

> **功能编号**: C2-F08  
> **功能名称**: 资产入库  
> **所属模块**: C2-资产管理 > 资产入库与管理  
> **主要角色**: SO (Software Owner)  
> **价值流阶段**: S3-方案设计

---

## 1. 功能概述

### 1.1 功能描述
将新开发的或外部引入的资产录入资产库，包括资产基本信息、技术文档、代码仓库、依赖关系等，建立完整的资产档案。

### 1.2 业务价值
- 沉淀研发成果，建立资产库
- 规范资产管理，提升资产质量
- 促进资产复用，提高研发效率

### 1.3 使用场景
- SO将新开发的模块入库为资产
- 团队将成熟的Feature入库为资产
- 引入外部优秀资产到资产库

## 3. 数据字段

### 3.1 基本信息

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| assetType | Enum | 是 | 资产类型: Product/Feature/Module |
| assetName | String | 是 | 资产名称 |
| assetCode | String | 是 | 资产编号(唯一) |
| productLine | Enum | 是 | 所属产品线: ADAS/CABIN/EE |
| description | String | 是 | 资产描述 |
| tags | String[] | 否 | 资产标签 |
| owner | String | 是 | 资产负责人ID |
| team | String[] | 否 | 资产团队成员ID列表 |

### 3.2 技术信息

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| techStack | String[] | 是 | 技术栈 |
| programmingLanguage | String[] | 是 | 编程语言 |
| framework | String[] | 否 | 使用的框架 |
| dependencies | Object[] | 否 | 依赖项列表 |
| codeRepository | String | 是 | 代码仓库URL |
| branch | String | 否 | 代码分支 |
| version | String | 是 | 资产版本 |
| performanceMetrics | Object | 否 | 性能指标 |

### 3.3 文档资料

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| designDoc | String | 否 | 设计文档URL |
| apiDoc | String | 否 | API文档URL |
| userManual | String | 否 | 使用手册URL |
| testReport | String | 否 | 测试报告URL |
| demoVideo | String | 否 | 演示视频URL |
| otherDocs | String[] | 否 | 其他文档URL列表 |

## 5. API接口

### 5.1 创建资产

**接口**: `POST /api/assets`

**请求示例**:
```json
{
  "assetType": "Module",
  "assetName": "车道线检测模块",
  "assetCode": "ASSET-ADAS-001",
  "productLine": "ADAS",
  "description": "基于深度学习的车道线检测模块...",
  "tags": ["ADAS", "视觉感知", "深度学习", "L2+"],
  "owner": "user-001",
  "team": ["user-002", "user-003"],
  "techStack": ["深度学习", "计算机视觉", "实时处理"],
  "programmingLanguage": ["Python", "C++"],
  "framework": ["TensorFlow", "OpenCV"],
  "dependencies": [
    {
      "name": "OpenCV",
      "version": "4.5.0",
      "required": true,
      "description": "图像处理库"
    }
  ],
  "codeRepository": "https://git.company.com/adas/lane-detection.git",
  "branch": "main",
  "version": "V2.0",
  "performanceMetrics": [
    {
      "name": "准确率",
      "target": ">95%",
      "actual": "97.5%",
      "description": "车道线检测准确率"
    }
  ],
  "designDoc": "https://docs.company.com/adas/lane-detection-design.pdf",
  "apiDoc": "https://docs.company.com/adas/lane-detection-api.html"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "assetId": "ASSET-ADAS-001",
    "status": "PENDING_REVIEW",
    "createdAt": "2025-01-15T10:00:00Z",
    "createdBy": "user-001"
  }
}
```

## 6. 交互设计

### 6.1 入库流程

1. **填写基本信息** (步骤1)
   - 选择资产类型
   - 填写资产名称、描述
   - 添加标签和负责人

2. **填写技术信息** (步骤2)
   - 填写技术栈、编程语言
   - 添加依赖项
   - 填写代码仓库和版本
   - 添加性能指标

3. **上传文档资料** (步骤3)
   - 上传设计文档
   - 上传API文档
   - 上传测试报告
   - 上传演示视频

4. **审核提交** (步骤4)
   - 预览资产信息
   - 提交审核
   - 等待审核结果
