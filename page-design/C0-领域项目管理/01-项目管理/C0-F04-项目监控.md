# C0-F04: 项目监控

> **功能编号**: C0-F04  
> **功能名称**: 项目监控  
> **主要用户**: PM（项目经理）、管理层、Scrum Master  
> **页面类型**: 仪表盘页  
> **优先级**: P1  
> **价值流阶段**: S4-S9（全流程）

---

## 一、功能概述

### 1.1 功能描述
提供整车项目的实时监控仪表盘，展示项目健康度、进度、风险、质量等关键指标，帮助PM和管理层快速掌握项目状态，及时发现和处理问题。

### 1.2 业务价值
- 实时掌握项目健康度
- 快速发现项目风险和问题
- 数据驱动的决策支持
- 提升项目透明度和可视化

### 1.3 使用场景
- PM每日查看项目状态
- 管理层定期检查项目健康度
- Scrum Master监控团队效能
- 项目评审会议展示项目进展

### 1.4 核心指标

**项目健康度指标**:
- 进度健康度：实际进度 vs 计划进度
- 质量健康度：缺陷率、测试覆盖率
- 资源健康度：团队负载率、资源利用率
- 风险健康度：风险数量、风险等级

**健康度评分规则**:
- 🟢 健康 (80-100分): 项目进展顺利
- 🟡 预警 (60-79分): 存在一定风险
- 🔴 异常 (0-59分): 需要立即关注

## 三、数据指标设计

### 3.1 健康度评分算法

**整体健康度** = (进度健康度 × 0.4) + (质量健康度 × 0.3) + (资源健康度 × 0.2) + (风险健康度 × 0.1)

**进度健康度**:
```
进度偏差 = (实际进度 - 计划进度) / 计划进度
进度健康度 = 100 - |进度偏差| × 100

评分规则:
- 进度偏差 ≤ 5%: 100分
- 进度偏差 5%-10%: 90-80分
- 进度偏差 10%-20%: 80-60分
- 进度偏差 > 20%: <60分
```

**质量健康度**:
```
质量健康度 = (测试覆盖率 × 0.4) + (缺陷密度评分 × 0.3) + (代码质量评分 × 0.3)

缺陷密度评分:
- 严重缺陷 = 0个: 100分
- 严重缺陷 1-3个: 80分
- 严重缺陷 4-6个: 60分
- 严重缺陷 > 6个: 40分
```

**资源健康度**:
```
资源健康度 = 100 - |平均负载率 - 75| × 2

最佳负载率: 70%-80%
- 负载率 70%-80%: 100分
- 负载率 60%-70% 或 80%-90%: 80分
- 负载率 50%-60% 或 90%-100%: 60分
- 负载率 < 50% 或 > 100%: 40分
```

**风险健康度**:
```
风险健康度 = 100 - (高风险数 × 20 + 中风险数 × 10 + 低风险数 × 5)

评分规则:
- 无风险: 100分
- 1-2个中低风险: 80-90分
- 1个高风险或3-5个中低风险: 60-79分
- 2个以上高风险: <60分
```

### 3.2 关键指标定义

| 指标名称 | 计算公式 | 数据来源 | 更新频率 |
|---------|---------|---------|---------|
| 项目进度 | 已完成Epic数 / 总Epic数 × 100% | Epic状态 | 实时 |
| Epic完成率 | 已完成Epic数 / 总Epic数 | Epic状态 | 实时 |
| 活跃风险数 | 状态为"活跃"的风险数量 | 风险管理 | 实时 |
| 待解决缺陷数 | 状态为"待解决"的缺陷数量 | 缺陷管理 | 实时 |
| 团队负载率 | 已分配工作量 / 团队容量 × 100% | 资源分配 | 每日 |
| 代码覆盖率 | 已覆盖代码行数 / 总代码行数 × 100% | CI/CD | 每次构建 |
| 缺陷密度 | 缺陷数 / 代码行数 × 1000 | 缺陷管理 | 每日 |
| 团队速率 | 实际完成Story点数 / 计划Story点数 × 100% | Sprint数据 | 每Sprint |

### 3.3 趋势分析

**进度趋势**:
- 横轴：时间（周/月）
- 纵轴：完成百分比
- 两条线：计划进度、实际进度
- 预警：实际进度低于计划进度5%以上

**燃尽图**:
- 横轴：时间（周）
- 纵轴：剩余工作量（Story点）
- 两条线：理想燃尽、实际燃尽
- 预警：实际燃尽高于理想燃尽10%以上

**质量趋势**:
- 横轴：时间（周）
- 纵轴：缺陷数量
- 多条线：严重缺陷、主要缺陷、次要缺陷
- 预警：严重缺陷数量上升

## 八、特殊说明

### 8.1 数据更新策略
- **实时数据**: 健康度、关键指标（每5分钟）
- **准实时数据**: 进度、风险、缺陷（每15分钟）
- **批量数据**: 趋势图、统计数据（每小时）
- **历史数据**: 保留最近90天的详细数据

### 8.2 性能优化
- 使用缓存减少API调用
- 图表数据按需加载
- 大数据量时使用分页
- 支持离线查看历史报告

### 8.3 权限控制
- PM：查看所有数据，导出报告
- 管理层：查看所有数据，导出报告
- Scrum Master：查看团队数据
- 团队成员：查看基本数据

### 8.4 预警规则
- 进度偏差 > 10%：发送邮件通知PM
- 高风险新增：发送即时通知
- 严重缺陷 > 5个：发送预警通知
- 团队负载 > 90%：发送预警通知
