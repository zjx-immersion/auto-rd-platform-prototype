## 一、功能概述

### 1.1 功能定位

MR-Team分配是模块设计的关键环节，负责将模块需求分配给开发团队，明确团队职责和交付物。

### 1.2 核心价值

- ✅ 明确团队职责
- ✅ 合理分配资源
- ✅ 平衡团队负载
- ✅ 支持并行开发
- ✅ 提高开发效率

### 1.3 需求流转

```
Epic → Feature → PRD → SSTS → MR → Task
                             ↑   ↑
                          P3步骤2.1 P3步骤2.3 (当前)
```

### 1.4 用户角色

- **主要用户**: PM、DL
- **协作用户**: SM、Architect
- **查看用户**: All

## 三、分配规则

### 3.1 技能匹配

1. **技能要求**: 根据MR的技能要求匹配团队
2. **技能评分**: 计算团队技能匹配度
3. **推荐团队**: 推荐技能匹配度最高的团队

### 3.2 负载均衡

1. **当前负载**: 统计团队当前负载
2. **预测负载**: 预测分配后的负载
3. **负载平衡**: 避免团队负载过高或过低

### 3.3 依赖关系

1. **前置依赖**: 考虑MR的前置依赖
2. **并行开发**: 支持无依赖的MR并行开发
3. **关键路径**: 识别关键路径上的MR

---

## 四、业务规则

### 4.1 分配规则

1. **一个MR一个团队**: 每个MR只能分配给一个团队
2. **团队负责人**: 必须指定团队负责人
3. **计划时间**: 必须指定计划开始和完成时间
4. **技能匹配**: 团队技能必须满足MR要求

### 4.2 AI智能分配

1. **技能匹配**: 优先匹配技能要求
2. **负载均衡**: 平衡团队负载
3. **依赖优化**: 优化依赖关系
4. **人工调整**: 支持人工调整
