## 🎯 使用指南

### 场景1: 查找功能页面设计

**步骤**:
1. 根据功能编号（如C1-F06）确定能力域
2. 进入对应能力域目录（如C1-需求管理/）
3. 查找对应功能的页面设计文档

**示例**:
```bash
# 查找C1-F06 PRD编写的页面设计
cd C2-方案设计/
cat C2-F00-PRD编写.md
```

### 场景2: 按价值流查找页面

**参考**: [全量功能清单_C0-C7.md 附录B](../platform-design/全量功能清单_C0-C7.md#附录b-按价值流阶段分类索引)

**价值流阶段映射**:
- **S1-S2: 需求分解** → C1-需求管理/
- **S3: 资产规划** → C2-资产管理/
- **S4: PI Planning** → C0/C3目录
- **S5: 迭代开发** → C4-迭代执行/
- **S6: 集成验证** → C5/C6目录
- **S7: 测试验收** → C5-质量保证/
- **S8: 制品晋级** → C6-发布管理/
- **S9: 产品交付** → C6-发布管理/

### 场景3: 按角色查找页面

**参考**: [全量功能清单_C0-C7.md 附录A](../platform-design/全量功能清单_C0-C7.md#附录a-按角色分类索引)

## 📝 页面设计规范

### 文档结构

每个页面设计文档应包含：

```markdown
# 功能编号-功能名称（如C1-F06-PRD编写）

## 1. 功能概述
- 功能描述
- 用户角色
- 使用场景

## 2. 页面结构
- 页面布局
- 主要区域
- 组件列表

## 3. 交互流程
- 操作流程
- 状态转换
- 异常处理

## 4. 数据模型
- 输入数据
- 输出数据
- API接口

## 5. UI设计
- 页面截图
- 组件样式
- 响应式布局

## 6. 技术实现
- 前端组件
- 状态管理
- API调用
```

### 设计原则

参考：[平台设计评估标准和原则.md](./平台设计评估标准和原则.md)

1. **用户中心**: 以用户体验为核心
2. **流程驱动**: 支持端到端业务流程
3. **数据准确**: 确保数据一致性
4. **性能优化**: 响应时间<2秒
5. **易用性**: 学习成本低、操作简单
